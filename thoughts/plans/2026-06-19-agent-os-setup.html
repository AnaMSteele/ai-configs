BDD Scenarios

Plan publication

Given the local service is running, when this file is registered with plan-review register, then the reviewer returns a browser URL and source sync remains active.

Skill load

Given Claude looks in ~/.claude/skills, when a future agent requests the standard workflow, then it can load Ana's linked canonical skill payload without copying or re-deriving behavior.

Origin alignment

Given local ai-configs changes exist, when an agent finishes work, then those changes are not done until they are committed and pushed to origin at AnaMSteele/ai-configs.

Central path blocked

Given ~/.agents is root-owned, when the canonical installer tries to create ~/.agents/skills, then the setup reports the blocker and uses Claude symlinks as a reversible local fallback.

Codex comment response

Given a reviewer leaves a pending browser comment, when plan-review-codex-responder.sh runs, then it claims one comment, gives the claim JSON and repo context to Codex, and expects Codex to update the plan or acknowledge the comment using the active claim.

Responder failure

Given Codex fails while handling a claimed comment, when the responder exits that attempt, then it releases the claim so the comment is not stranded.
